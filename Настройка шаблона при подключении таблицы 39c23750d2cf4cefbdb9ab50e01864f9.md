# Настройка шаблона при подключении таблицы

Если в качестве источника данных вы выбрали **табличный файл**, то для корректного считывания позиций и атрибутов необходимо заполнить шаблон. Если при создании нового подключения вы пропустили этот шаг, настроить шаблон можно во вкладке “Настройки” → “Шаблон файла”

![Untitled](%D0%9D%D0%B0%D1%81%D1%82%D1%80%D0%BE%D0%B9%D0%BA%D0%B0%20%D1%88%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD%D0%B0%20%D0%BF%D1%80%D0%B8%20%D0%BF%D0%BE%D0%B4%D0%BA%D0%BB%D1%8E%D1%87%D0%B5%D0%BD%D0%B8%D0%B8%20%D1%82%D0%B0%D0%B1%D0%BB%D0%B8%D1%86%D1%8B/Untitled.png)

- **rows** - количество строк в шаблоне
- **columns** - количество столбцов в шаблоне

Шаблон создается из расчета, что каждая следующая группа строк повторяет шаблон.

![Untitled](%D0%9D%D0%B0%D1%81%D1%82%D1%80%D0%BE%D0%B9%D0%BA%D0%B0%20%D1%88%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD%D0%B0%20%D0%BF%D1%80%D0%B8%20%D0%BF%D0%BE%D0%B4%D0%BA%D0%BB%D1%8E%D1%87%D0%B5%D0%BD%D0%B8%D0%B8%20%D1%82%D0%B0%D0%B1%D0%BB%D0%B8%D1%86%D1%8B/Untitled%201.png)

Для примера таблицы выше шаблон может выглядеть следующим образом:

![Untitled](%D0%9D%D0%B0%D1%81%D1%82%D1%80%D0%BE%D0%B9%D0%BA%D0%B0%20%D1%88%D0%B0%D0%B1%D0%BB%D0%BE%D0%BD%D0%B0%20%D0%BF%D1%80%D0%B8%20%D0%BF%D0%BE%D0%B4%D0%BA%D0%BB%D1%8E%D1%87%D0%B5%D0%BD%D0%B8%D0%B8%20%D1%82%D0%B0%D0%B1%D0%BB%D0%B8%D1%86%D1%8B/Untitled%202.png)

Первая строка в таблице - названия столбцов, она не является товаром, который нужно считывать и заносить в каталог, поэтому в поле “Пропустить строчки” установлена единица. Информация об одном товаре занимает одну строку и 4 столбца, поэтому соответствующие поля заполнены этими значениями. Далее в теле самого шаблона прописаны говорящие названия каждого из атрибута товара, согласно загружаемой таблице. 

❗ Обратите внимание, что клетка “Цена” у одного из товаров пуста, в связи с этим, в шаблоне перед соответвующим атрибутом, необходимо поставить “!” (восклицательный знак). Восклицательный знак надо поставить перед каждым атрибутом значение которого может быть не задано, в противном случае Databird не сможет правильно считать таблицу.
