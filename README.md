# Руководство по статьям (web-only)

Вся работа выполняется через веб‑интерфейс GitHub и всегда напрямую в ветку `main`.

Разделы:
- Общие принципы
- Создание новой статьи
- Правка существующей статьи

---

## Общие принципы

- Все статьи находятся в папке `docs/` как отдельные папки: `docs/<slug>/`.
- В каждой папке должен быть файл `index.md` с YAML front matter, например:

```yaml
---
title: "Заголовок статьи"
---
```

- Правила для `slug`: только латинские строчные буквы, цифры и дефисы (`-`).
- Ресурсы (изображения, файлы) кладите в ту же папку и используйте относительные ссылки, например: `![alt](./image.png)`.
- Для синтаксиса Markdown используйте руководство GitHub: https://docs.github.com/en/get-started/writing-on-github/basic-writing-and-formatting-syntax

## Создание новой статьи

Порядок действий (через веб‑интерфейс GitHub, всегда в `main`):

1. Создайте папку с `slug` именем и файл `index.md`: в **Add file → Create new file** введите `my-new-article-slug/index.md`.
2. Создайте `index.md` (шаг 1 создаёт файл автоматически).
3. Напишите статью в `index.md` с использованием синтаксиса GitHub Markdown (ссылка выше).
4. Добавьте ресурсы в ту же папку через **Add file → Upload files** и укажите относительные пути в статье.
5. Обновите корневой `index.md`: откройте `docs/index.md`, нажмите **Edit** и добавьте ссылку в формате `[Заголовок](/my-new-article-slug/)`, затем внизу укажите сообщение коммита и нажмите **Commit directly to the main branch**.

Пример минимального `index.md`:

```markdown
---
title: "Мой новый материал"
---

# Мой новый материал

Текст статьи...
```

## Правка существующей статьи

- Откройте `docs/<slug>/index.md` на GitHub и нажмите значок карандаша (Edit).
- Внесите изменения, проверьте в **Preview**.
- Для добавления изображений используйте **Add file → Upload files** в той же папке.
- Внизу укажите сообщение коммита и нажмите **Commit directly to the main branch** — всегда работаем в `main`.
